Why Helm in real companies
--------------------------
Raw yaml folders get messy. **Helm chart** = templated package (Deployment, Service,
Ingress, HPA in one bundle). **values.yaml** changes per environment (dev/uat/prod).


Helm concepts
-------------
  Chart    — package (templates + values)
  Release  — installed instance of a chart
  Repo     — chart repository (like helm repo add bitnami)


Install Helm CLI (on EC2 / laptop)
----------------------------------
  curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
  helm version


Commands I practiced today
--------------------------
  helm repo add bitnami https://charts.bitnami.com/bitnami
  helm repo update
  helm search repo nginx

  helm install my-nginx bitnami/nginx --namespace dev --create-namespace
  helm list -A
  helm status my-nginx -n dev
  kubectl get pods -n dev

Upgrade (new config / image):
  helm upgrade my-nginx bitnami/nginx -n dev --set service.type=NodePort
  helm history my-nginx -n dev

Rollback:
  helm rollback my-nginx 1 -n dev

Uninstall:
  helm uninstall my-nginx -n dev

Dry-run (before prod):
  helm install my-nginx bitnami/nginx -n dev --dry-run --debug


Multi-environment pattern (class)
---------------------------------
  values-dev.yaml    — 1 replica, small resources
  values-prod.yaml   — 3 replicas, probes on, ingress enabled

  helm upgrade --install app ./my-chart -f values-dev.yaml -n dev
  helm upgrade --install app ./my-chart -f values-prod.yaml -n prod


Custom chart (mini project)
---------------------------
  helm create pathnex-app
  # edit templates/deployment.yaml, values.yaml
  helm lint pathnex-app
  helm install pathnex pathnex-app -f values-dev.yaml
  helm upgrade pathnex pathnex-app -f values-prod.yaml


Cluster upgrade (Class 5 topic — high level)
--------------------------------------------
  - Control plane version upgraded in cloud (EKS console / eksctl)
  - Node groups drained and cycled to new kubelet version
  - Check API deprecation warnings before jump versions
  (We did not run production upgrade in lab — theory + AWS docs only.)


Interview mistakes to avoid (Helm + K8s)
----------------------------------------
  - Editing live resources but not chart → next helm upgrade overwrites
  - Secrets in values.yaml committed to Git
  - No resource limits → noisy neighbor on node
  - Using latest tag instead of pinned image digest in prod


Quick kubectl still used with Helm
----------------------------------
  kubectl get all -n dev
  kubectl logs -l app.kubernetes.io/instance=my-nginx -n dev
  helm get values my-nginx -n dev
