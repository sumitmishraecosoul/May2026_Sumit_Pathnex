Linux — Day 01 (02 May 2026)
PathNex class notes + what I practiced today
========================================

Today's class was the first Linux session. We started from how the terminal
actually works on a server, then spent most of the time moving around the
filesystem and working with files. I wrote these notes right after class so
I remember what I ran and why.


What I understood today
-------------------------
Linux on our VMs is what we SSH into — not something separate from "the server".
The shell (bash) reads my command and runs programs. Paths always start from /.

Folders I kept seeing in class:
  /home/<user>   my home (~)
  /etc           configs
  /var/log       logs (we only peeked today)
  /tmp           temporary stuff

Small habits that help:
  Tab completes paths/names
  man <cmd>      full help
  <cmd> --help   quick help


Commands I used / practiced today
-----------------------------------

Moving around:
  pwd
  cd /path
  cd ~
  cd -

Listing files (I used -l and -a a lot today):
  ls
  ls -l
  ls -la
  ls -lh

Creating / changing files:
  mkdir dirname
  mkdir -p parent/child/grandchild
  touch newfile.txt
  cp file1 file2
  cp -r folder1 folder2
  mv oldname newname
  rm file
  rm -r folder    (class reminder: double-check path before -r)

Reading files:
  cat smallfile.txt
  less bigfile.log     (q to quit)
  head -n 10 file
  tail -n 10 file

find — I used this to search by name under a directory:
  find . -name "*.txt"
  find /var/log -name "*.log"
  find . -type f -mtime -7

grep — search inside text / filter output:
  grep "error" some.log
  grep -i "failed" app.log
  grep -rn "listen" /etc/nginx/
  grep -n "ERROR" app.log
  ps aux | grep nginx

awk — pull columns (still new for me, did a few examples):
  ps aux | awk '{print $1, $11}'
  awk -F: '{print $1}' /etc/passwd

sed — replace text on a line (practiced on a test file):
  sed 's/debug/info/g' test.txt
  sed -n '5,10p' test.txt
