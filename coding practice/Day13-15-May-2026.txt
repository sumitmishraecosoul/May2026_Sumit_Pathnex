Pods deeply
-----------
Pod = one or more containers, shared IP, scheduled together.
Pods are ephemeral — IP changes on restart → Services fix stable access later.


Labels & selectors (how traffic finds pods later)
-------------------------------------------------
  labels:      key=value tags on objects
  selectors:   query those labels

  kubectl get pods --show-labels
  kubectl get pods -l app=nginx
  kubectl label pod nginx tier=frontend
  kubectl label pod nginx env- --overwrite   # remove label key env if needed


Commands I practiced today
--------------------------
Create / inspect:
  kubectl apply -f pod.yaml
  kubectl get pods -w
  kubectl get pod <name> -o yaml
  kubectl describe pod <name>

Debug (interview favorites):
  kubectl logs <pod>
  kubectl logs <pod> -c <container>    # multi-container pod
  kubectl logs <pod> --previous        # crashed container
  kubectl exec -it <pod> -- /bin/bash
  kubectl exec <pod> -- env

Events & status:
  kubectl get events --sort-by=.metadata.creationTimestamp
  kubectl describe pod <name> | grep -A5 Events

Delete:
  kubectl delete pod <name>
  kubectl delete -f pod.yaml

Example pod.yaml we applied:
  apiVersion: v1
  kind: Pod
  metadata:
    name: web
    labels:
      app: web
  spec:
    containers:
    - name: nginx
      image: nginx:1.25
      ports:
      - containerPort: 80


Troubleshoot flow from class
----------------------------
  1. kubectl get pods          — Status? Ready?
  2. kubectl describe pod      — Events at bottom
  3. kubectl logs              — app error
  4. kubectl exec              — curl localhost inside pod
