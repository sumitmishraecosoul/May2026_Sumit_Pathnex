What Docker is (simple)
-----------------------
Docker packages an app + its dependencies into a **container** — a small isolated
environment that runs the same on my laptop, EC2, or production.

Companies use it because:
  - dev/test/prod all behave the same
  - starts fast compared to full VMs
  - fits CI/CD (build image once, run anywhere)
  - each microservice can run in its own container


Image vs container (clear difference)
-------------------------------------
  **Image**  = read-only recipe/template (like a class blueprint)
  **Container** = running instance of that image (like a student actually doing the lab)

One image → many containers. Delete container → image still on disk.


Dockerfile basics (build your own image)
----------------------------------------
Example from class (Node app):
  FROM node:14
  WORKDIR /user/pathnex
  COPY . .
  RUN npm install
  EXPOSE 3000
  CMD ["npm", "start"]

Instructions mean:
  FROM     — base image to start from
  WORKDIR  — directory inside image
  COPY     — add my code
  RUN      — install deps at build time
  EXPOSE   — document port
  CMD      — what runs when container starts


Docker Engine (what runs containers)
------------------------------------
  - **docker** CLI — what I type
  - **daemon** — background (class noted many setups now use **containerd** under the hood)
  - REST API — tools/automation talk to Docker


Install Docker on Ubuntu (EC2 lab)
----------------------------------
  sudo apt update
  sudo apt install docker.io -y
  sudo systemctl enable --now docker
  docker --version
  sudo systemctl status docker
  # optional: sudo usermod -aG docker $USER   then re-login


Commands I used today (must-know for jobs)
-------------------------------------------

Images from registry:
  docker pull nginx
  docker images

Run containers:
  docker run -it nginx /bin/bash          # interactive shell, exit stops container
  docker run -d --name nginx-container nginx   # detached background
  docker ps                               # running only
  docker ps -a                            # including stopped

Lifecycle:
  docker stop nginx-container
  docker start nginx-container
  docker rm nginx-container               # remove container (stopped first)
  docker rmi nginx                        # remove image if not needed

Build my own image:
  docker build -t nginx-image .
  docker tag nginx-image myuser/nginx-image:v1

Inspect / debug:
  docker inspect nginx-container
  docker logs nginx-container
  docker exec -it nginx-container /bin/bash
