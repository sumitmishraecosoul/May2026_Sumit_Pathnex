Helm — what it is (Class 5 start)
---------------------------------
Helm = **package manager for Kubernetes** (like apt for Ubuntu).

Instead of applying 10 separate yaml files by hand, one **chart** installs
Deployment + Service + Ingress + ConfigMap together.

Key pieces:
  **Helm client**     — helm CLI on my laptop/EC2
  **Chart**           — folder: Chart.yaml + values.yaml + templates/
  **Repository**      — where charts are stored (Helm Hub / bitnami)
  **Release**         — one installed chart instance in the cluster

Helm v3 — **no Tiller** (old v2 server is gone). Only need kubectl + helm CLI.


Why companies use Helm
----------------------
  - faster repeatable deploys
  - versioned upgrades and rollbacks
  - values.yaml per environment (dev / staging / prod)
  - share charts across teams


How Helm works (class diagram in notes)
---------------------------------------
  1. Chart has templates + default values
  2. helm install renders templates → Kubernetes yaml → applies to cluster
  3. helm upgrade changes release; helm rollback if broken


Install Helm (EC2 / Linux)
--------------------------
  curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
  helm version
  # macOS class note: brew install helm
  # Helm v3: just verify kubectl context is correct


Repo + first install (class examples)
-------------------------------------
  helm repo add stable https://charts.helm.sh/stable
  helm repo add bitnami https://charts.bitnami.com/bitnami
  helm repo update
  helm search repo nginx

  # class example (stable repo — lab also used bitnami):
  helm install my-nginx-ingress stable/nginx-ingress

  # what I ran on lab (bitnami):
  helm install my-nginx bitnami/nginx --namespace pathnex --create-namespace

  kubectl get pods -n pathnex
  helm list -n pathnex
  helm status my-nginx -n pathnex
